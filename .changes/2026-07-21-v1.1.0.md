# Migration → 1.1.0 (2026-07-21)

**Wording only. No structural change.** Nothing moves, no lint rule changes, no manifest changes — a
drop-in upgrade. This release fixes a discoverability trap in the always-loaded `.knowledge/README.md`: the
homes table described `scripts/` as only "the linter + its teeth-test," and the Versioning note said "Never
edit those files," so an agent looking for a home for a project check would rule `scripts/` out and invent a
new top-level directory (`bin/`) — never opening `scripts/README.md`, which already says the folder is also
its workspace. The affordance existed but was hidden behind the signal that steered agents away from it.

## What changed

- **`.knowledge/README.md` homes table** — the `scripts/` row now reads *"Tooling — the versioned linter,
  plus your own project scripts,"* surfacing the workspace affordance in the always-loaded map.
- **`.knowledge/README.md` Versioning note** — adds one clause: *"Your own project scripts live beside them
  in `scripts/` — only the listed files are off-limits,"* so "Never edit those files" can't be misread as
  "never add your own."
- **`.knowledge/README.md` homes table `tmp/` row** — now reads *"Git-ignored scratch — throwaway working
  files, never committed,"* completing the contrast with `scripts/` so the durable-vs-throwaway choice is
  unambiguous at a glance (a committed, CI-wired check goes in `scripts/`, not `tmp/`).

Only `README.md` changed. The `guides/docs-*.md` standards, the two shipped scripts, and the
`.payload-manifest` are untouched, so integrity checksums still match.

## Migration (apply in order)

1. **Replace `.knowledge/README.md`** with the 1.1.0 copy from `template/.knowledge/README.md` (or apply the
   three wording edits above by hand). This is the only content change.
2. **Stamp.** Set `.knowledge/.version` to `1.1.0`.
3. **Lint.** Run `python3 .knowledge/scripts/doc-lint .knowledge` — it must stay green (the manifest is
   unchanged, so integrity passes without a regenerate).

Project-specific scripts you have already placed elsewhere (e.g. a top-level `bin/`) may move into
`.knowledge/scripts/` alongside `doc-lint` — `doc-lint` tolerates extra files there and never versions them.
This is optional cleanup, not required by the upgrade.

## Verify

- `doc-lint` exits clean; `test_doc_lint.py` still passes every case (unchanged — no rule moved).
- `.knowledge/README.md`'s `scripts/` row names both the linter and your own scripts; the Versioning note
  says only the listed files are off-limits; the `tmp/` row marks it throwaway and never committed.
- `.knowledge/.version` reads `1.1.0`.
- The `.payload-manifest` is unchanged and integrity passes (no versioned file was touched).
