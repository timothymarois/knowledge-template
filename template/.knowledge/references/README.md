# references/ — visual targets (catalog)

The **catalog** of visual targets — screenshots, competitor captures, concept art, UI we want to match.
The images are the point; a note beside each says what to take and what to avoid. Sibling of
[`../research/`](../research/) (research is the *analysis*, references are the *targets*), used at any
stage: researching, designing, or building toward a look.

## How to add one

- **Organize by competitor or platform first** — a folder per source (`stripe/`, `linear/`, `ios/`,
  `figma/`) — then by what you're comparing inside it (a screen, a flow, a style). Grouping by source is
  what lets us compare *the same surface across competitors* and see how each solved it.
- A `NOTES.md` inside each set: what inspires us here, what to avoid, and provenance for every image.
- **PII-free.** Blur or crop personal data out of captures. Respect licensing.

Cite a target from a `../research/` note or a `../prd-drafts/` proposal — never copy the analysis here.
